# Playwright 테스트 자동화 룰

## 코드 스타일
- TypeScript 사용 필수
- 모든 테스트는 `test()` 함수로 작성
- 테스트 파일은 `.spec.ts` 확장자 사용
- 페이지 객체 패턴(Page Object Model) 사용 권장

## 테스트 작성 규칙
- 각 테스트는 독립적으로 실행 가능해야 함
- 테스트 전후에 적절한 setup/teardown 사용
- 명확한 테스트 이름 사용 (한글 또는 영어)
- Given-When-Then 패턴 준수

## 에러 처리
- 모든 네트워크 요청에 대한 타임아웃 설정
- 실패한 테스트는 스크린샷 자동 저장
- 에러 메시지는 명확하고 디버깅 가능해야 함

## 베스트 프랙티스
- 하드코딩된 대기 시간 대신 `waitFor` 사용
- 선택자는 data-testid 속성 우선 사용
- 테스트 데이터는 별도 파일로 관리
- 환경 변수로 설정값 관리
